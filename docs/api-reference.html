外部スポットサービス連携の使い方

TripLog の API は、Django のローカルDBと外部スポットAPIサービスのどちらからでもデータを取得できます。 以下の手順で環境変数を設定すると、アプリケーション全体が外部サービス経由のモードに切り替わります。 既定ではローカルDBモードが有効です。

ローカルDBモード

SPOT_API_USE_EXTERNAL を false または未設定にすると、従来どおり Django のモデルからスポット情報を読み書きします。

外部APIモード

SPOT_API_USE_EXTERNAL を true にし、SPOT_API_BASE_URL などを設定すると、全てのスポットAPIが外部サービスに委譲されます。

環境変数・設定一覧

名前 必須 役割 既定値 / 例
SPOT_API_USE_EXTERNAL 任意 外部APIモードを有効化（true / false） 未設定時は false
SPOT_API_BASE_URL 外部モード時は必須 外部サービスのベースURL（末尾スラッシュ不要） 例: https://spots.example.com
SPOT_API_TIMEOUT 任意 外部API呼び出しのタイムアウト秒 既定値: 5.0
SPOT_API_TOKEN 任意 APIトークン。設定すると自動で認証ヘッダーを追加 例: my-secret-token
SPOT_API_TOKEN_HEADER 任意 トークンを送るHTTPヘッダー名 既定値: Authorization
SPOT_API_SEARCH_PATH 任意 検索エンドポイントのパス 既定値: /api/search/
SPOT_API_LIST_PATH 任意 一覧取得エンドポイントのパス 既定値: /api/spots/
SPOT_API_CREATE_PATH 任意 スポット作成エンドポイントのパス 既定値: /api/spots/add/

推奨 .env サンプル

SPOT_API_USE_EXTERNAL=true
SPOT_API_BASE_URL=https://spots.example.com
SPOT_API_TOKEN=your-api-token
SPOT_API_TIMEOUT=8.0
SPOT_API_SEARCH_PATH=/v1/search/
SPOT_API_LIST_PATH=/v1/spots/
SPOT_API_CREATE_PATH=/v1/spots/add/

導入ステップ

  1. 外部スポットサービスの契約・発行済みAPIキーとベースURLを確認します。
  2. 上記の環境変数を `.env` やデプロイ先の設定画面に登録し、アプリを再起動します。
  3. 開発環境では `python manage.py test` や `curl` を利用し、外部APIにアクセスできることを確認します。
  4. 万一切り戻したい場合は `SPOT_API_USE_EXTERNAL=false` に変更し、再起動後にローカルDBモードへ戻します。

動作確認用 cURL 例

一覧取得

curl -X GET "https://your-domain.com/api/spots/" \
  -H "Authorization: Bearer LOCAL_OR_PROXY_TOKEN"

外部API直アクセス

curl -X GET "https://spots.example.com/api/spots/" \
  -H "Authorization: Bearer your-api-token"

トラブルシューティング

  • 「外部スポットAPIへの接続に失敗しました」: ネットワークや `SPOT_API_BASE_URL` を確認し、タイムアウト値を調整します。
  • 「レスポンス形式が不正です」: 外部APIのレスポンスが JSON 形式であるか、キー名が `spots` / `results` になっているかを確認します。
  • 外部APIからのエラーコードがそのまま返却されるため、クライアント側で 4xx / 5xx をハンドリングしてください。

GET /api/spots/

説明

登録されている全スポットの情報をJSON形式で取得します。オプションでフィルタリングが可能です。

クエリパラメータ

パラメータ 型 必須 説明
filter string 任意 フィルター種別（mine: 自分の投稿, others: 他者の投稿）

レスポンス例

{
  "spots": [
    {
      "id": 1,
      "title": "東京駅",
      "description": "日本の玄関口として知られる...",
      "latitude": 35.681236,
      "longitude": 139.767125,
      "address": "東京都千代田区丸の内1丁目",
      "image": "https://example.com/image.jpg",
      "created_by": "user1",
      "created_at": "2025-01-01T10:00:00Z",
      "tags": ["駅", "観光地"]
    }
  ]
}

cURL 例

curl -X GET "https://your-domain.com/api/spots/" \
  -H "Authorization: Bearer YOUR_TOKEN"

POST /api/spots/add/

説明

新しいスポットを登録します。認証が必要です。

リクエストボディ

フィールド 型 必須 説明
title string 必須 スポット名
description string 必須 スポットの説明
latitude float 必須 緯度
longitude float 必須 経度
address string 任意 住所
image file 任意 画像ファイル
image_url string 任意 画像URL

レスポンス例

{
  "success": true,
  "spot": {
    "id": 123,
    "title": "新しいスポット",
    "description": "説明文...",
    "latitude": 35.681236,
    "longitude": 139.767125,
    "address": "住所",
    "image": null,
    "created_by": "current_user",
    "created_at": "2025-01-01T10:00:00Z"
  }
}

cURL 例

curl -X POST "https://your-domain.com/api/spots/add/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=東京タワー" \
  -F "description=東京のシンボル..." \
  -F "latitude=35.658034" \
  -F "longitude=139.745432" \
  -F "address=東京都港区芝公園4丁目2-8"

GET /api/search/

説明

キーワードによるスポット検索を行います。タイトル、説明、住所、タグが検索対象です。

クエリパラメータ

パラメータ 型 必須 説明
q string 必須 検索キーワード

レスポンス例

{
  "results": [
    {
      "id": 1,
      "title": "東京駅",
      "address": "東京都千代田区丸の内1丁目",
      "latitude": 35.681236,
      "longitude": 139.767125
    },
    {
      "id": 2,
      "title": "東京タワー",
      "address": "東京都港区芝公園4丁目2-8",
      "latitude": 35.658034,
      "longitude": 139.745432
    }
  ]
}

cURL 例

curl "https://your-domain.com/api/search/?q=東京"

POST /accounts/login/

説明

ユーザー認証を行い、セッションを確立します。Djangoの標準認証システムを使用しています。

リクエストボディ

フィールド 型 必須 説明
username string 必須 ユーザー名
password string 必須 パスワード

レスポンス例

# 成功時: 302 リダイレクト
Location: /

# 失敗時: 200 OK + エラーメッセージ
{
  "error": "Invalid credentials"
}

エラーレスポンス

HTTP ステータスコード

  • 200 成功
  • 400 リクエストエラー
  • 401 認証エラー
  • 403 権限エラー
  • 404 リソース未検出
  • 500 サーバーエラー

エラー例

{
  "success": false,
  "error": "タイトルと説明は必須です。"
}
