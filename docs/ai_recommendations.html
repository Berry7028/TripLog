機能概要

TripLogのAIおすすめ機能は、ユーザーの過去の閲覧履歴を分析し、個々の興味・嗜好に合ったスポットを自動的に推薦する革新的なシステムです。 単なる人気ランキングとは異なり、各ユーザーの行動パターンをAIが学習し、パーソナライズされたおすすめを提供します。

この機能により、ユーザーは自分の興味に合った新しいスポットを発見しやすくなり、アプリの利用満足度が向上します。 また、開発者にとっては、複雑な推薦アルゴリズムを自前で実装する必要がなく、AIの力を借りて高品質な推薦を実現できます。

AI分析

OpenRouter API (GPT-4o-mini) を使用したユーザーの興味分析

スコアリング

0-100のスコアでスポットの関連度を数値化

バックグラウンド処理

定期的なバッチ処理で全ユーザーのスコアを事前計算

実装方法

AIおすすめ機能の実装は、主に3つのコンポーネントから構成されています。 まず、ユーザーの閲覧行動を追跡・蓄積するデータ収集レイヤー、次にAIによる分析を行う推論レイヤー、 最後に計算結果を保存・活用するストレージレイヤーです。

このアーキテクチャにより、リアルタイムでの推薦が可能でありながら、 バックグラウンドでのバッチ処理によってシステム全体のパフォーマンスを最適化しています。

データモデル

UserSpotInteraction

ユーザーのスポット閲覧データを蓄積するモデル

user = models.ForeignKey(User, on_delete=models.CASCADE)
spot = models.ForeignKey(Spot, on_delete=models.CASCADE)
view_count = models.PositiveIntegerField(default=0)
total_view_duration = models.DurationField(default=timedelta)
last_viewed_at = models.DateTimeField(auto_now=True)

UserRecommendationScore

計算されたおすすめスコアを保存するモデル

user = models.ForeignKey(User, on_delete=models.CASCADE)
spot = models.ForeignKey(Spot, on_delete=models.CASCADE)
score = models.FloatField(default=0.0)
source = models.CharField(max_length=20, default='api')
reason = models.TextField(blank=True)

処理フロー

1

ユーザーの閲覧履歴収集

UserSpotInteraction からユーザーの閲覧データ（回数、滞在時間、最終閲覧日時、タグ）を収集

2

AI分析リクエスト

OpenRouter API にユーザーのインタラクション履歴と全スポット情報を送信

3

スコア計算

AIが各スポットの関連度を0-100のスコアで評価

4

結果保存

UserRecommendationScore にスコアを保存し、スポット一覧の並び替えに使用

スコアリングロジック

スポットのおすすめ度を数値化するために、0から100までのスコアを使用しています。 100に近いほどユーザーの興味・嗜好に合致していることを示し、0に近いほど関連性が低いことを意味します。

スコアリングは主にAIによる分析を優先しますが、APIが利用できない場合やエラーが発生した場合は、 ルールベースのフォールバック計算に自動的に切り替わります。これにより、システムの堅牢性を確保しています。

AIプロンプト

AIに与えられる指示と分析基準

あなたは旅行アプリのレコメンドAIです。ユーザーの閲覧履歴を受け取り、
各スポットの関連度スコア(0〜100)を JSON 形式で返してください。

重要な指示:
1. ユーザーが過去に閲覧したスポット(interactions)の傾向を分析してください。
2. 全スポットリスト(target_spots)には、ユーザーがまだ見ていないスポットも含まれています。
3. 閲覧済みスポットと類似のタグ、説明、テーマを持つ未閲覧スポットは、高いスコアを付けてください。
4. ユーザーの興味・嗜好に合致する新しい発見となるスポットを積極的に推薦してください。
5. 出力形式: {"scores": [{"spot_id": number, "score": number, "reason": string}]}

レスポンス形式

AIからのスコア結果の期待される形式

フォールバックスコアリング

APIが利用できない場合のルールベースのスコア計算

閲覧回数ボーナス

view_count × 2.0

直近性ボーナス

max(0.0, 3.0 - (経過日数 / 10.0))

30日以内の閲覧に重み付け

滞在時間ボーナス

min(滞在時間(分), 60.0) × 0.5

最大60分まで評価

計算式

total_score = view_bonus + recency_bonus + duration_bonus

API連携

AIおすすめ機能の心臓部であるOpenRouter APIとの連携は、環境変数によって柔軟に設定可能です。 これにより、本番環境と開発環境で異なるAPIキーやモデルを使用することができます。

また、バックグラウンドジョブによる定期実行により、ユーザーの閲覧行動の変化に追従した推薦を実現しています。 ジョブの実行間隔は管理画面から調整可能で、システムの負荷と推薦の鮮度のバランスを最適化できます。

環境変数設定

OPENROUTER_API_KEY

OpenRouter APIの認証キー

OPENROUTER_RECOMMENDATION_URL

APIエンドポイントURL (デフォルト: openrouter.ai)

OPENROUTER_RECOMMENDATION_MODEL

使用モデル (デフォルト: openai/gpt-4o-mini)

OPENROUTER_TIMEOUT

APIタイムアウト秒数 (デフォルト: 15)

バックグラウンドジョブ

定期実行設定

RecommendationJobSetting で実行間隔を管理

interval_hours = models.PositiveIntegerField(default=31)
enabled = models.BooleanField(default=True)
last_run_at = models.DateTimeField(blank=True, null=True)

実行コマンド

Django management command で実行

python manage.py run_recommendation_jobs
